## 💡 Using Your Prompts

Each file in this kit serves a specific purpose in your creative chain. 

### The Project Brief
Start with the project brief. Open the text file named brief-template.txt. This document outlines the 2.5D style you want. It defines how elements move when the user scrolls. Copy the text from this file.

### The AI Prompt
Paste the text from the project brief into your chosen AI tool. You may use tools like Claude or other prompt-based coding assistants. The AI reviews the instructions. It generates the layout and the scrolling logic for your website. 

### Refining Results
The prompt includes tokens specifically for visual depth. If you want more motion, adjust the scroll-speed variables in the prompt. You do not need to change the technical code. You only change the plain English instructions within the prompt.

## 🛠️ Troubleshooting Common Issues

If the AI produces an error, check these items:

*   **Character Limits:** Some AI tools have a word count limit. If your prompt is too long, break it into two parts. 
*   **Context Loss:** If the AI forgets the project style, repeat the initial project brief.
*   **Browser Locks:** If the animations look stuttered, clear your browser cache. This forces the browser to load the new scroll logic.
*   **File Permissions:** If Windows blocks the files, right-click the folder, go to Properties, and select Unblock at the bottom of the window.

## 📈 Tips for Better Animations

Keep your animations smooth by following these rules:

*   **Limit Layers:** Keep your 2.5D depth to three or four layers. Too many layers make the page load slowly.
*   **Consistent Speed:** Use consistent scroll tags across your different pages. This makes the site feel like one cohesive piece.
*   **File Sizes:** Ensure your images are small before you use them. Large images hurt the scroll performance.
*   **Testing:** Test your site on a phone and a computer. The scroll logic behaves differently on touch screens.
